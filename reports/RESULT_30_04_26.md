## Failure Analysis

### F1 — Recall Breadth: Multi-Session Aggregation (c004, c010)

**Cases:** c004 (magazine subscriptions), c010 (charity event a month ago)  
**Session recall@5:** 0.5 for both — only 2/4 gold sessions reached.  
**Root cause:** RRF + CE reranker concentrates top ranks on the most semantically similar session. Other sessions containing equally relevant facts are ranked lower and fall outside the effective window even at top_k=25.  
**Impact:** Missing session → missing facts → wrong count or wrong event selection.  
**Potential fix:** Increase top_k further, or implement session-aware re-ranking (ensure at least 1 result per session that matches the query).

### F2 — Knowledge-Update Temporal Ordering (c006, partial c007)

**Cases:** c006 (sneakers location), c007 (mortgage amount)  
**Root cause:** When two sessions contain conflicting values for the same entity, the generation model has no mechanism to determine which session is more recent — session timestamps are not exposed in the generation context.  
**c005 succeeded** because one memory happened to include an explicit ISO timestamp (`2026-04-29T09:47:58`) in its text, allowing Minimax-M2.7 to reason about recency.  
**c006 failed** because both location facts were bare text ("under the bed", "in a shoe rack in closet") with no session timestamp visible.  
**c007 partial**: model correctly flagged the conflict instead of confidently picking wrong — but gold answer was $400,000 and the "correct" (newer) value could not be determined.  
**Potential fix:** Expose session ingestion timestamp (or session sequence number) in the generation context for every recalled fact. This enables the model to always prefer the most recent session.

### F3 — Temporal Absolute Date Awareness (c011, contributing to c010)

**Cases:** c011 (smoker purchase date), c010 (charity event timing)  
**c011:** Memory extracted "purchased smoker **today**" at ingestion time. The question was asked ~10 days later. The model sees "today" in memory and reports "purchased today" — it cannot compute elapsed time between ingestion date and question date.  
**c010:** "Walk for Hunger" is labeled "today" (April 30, 2026). The question "a month ago" is asked from ~May 30. The model sees "today" and interprets the event as current, not one month old.  
**Root cause:** Relative time expressions ("today", "recently") in extracted facts become stale after the ingestion date. The model has no anchor to resolve "today" against the current date.  
**Potential fix:** Store `ingested_at` absolute timestamp alongside extracted facts and include it in the generation context prompt. Allow the model to compute: `"today" in memory → ingested_at date` and then reason about elapsed time from current date.

## Retain Quality Notes

- **2-pass extraction confirmed** across all banks (chunk IDs use `p2_*` suffix).
- **c000:** All 5 model kit experience facts successfully extracted by 2-pass (Tiger I at rank 21 in top-25, within retrieval window).
- **c001:** Chili pepper session facts extracted but not surfaced in top-25 recall (recall breadth issue, not extraction issue).
- **c004:** New Yorker and Forbes subscriptions extracted correctly. A second active subscription exists in the bank but its session is ranked below top-25.
- **c006, c007:** Both conflicting sessions extracted correctly; failure is in generation-side temporal ordering, not retain.
- **c010, c011:** Temporal expressions ("today", "recently") extracted as-is without absolute anchoring — intentional by current design, but limits post-ingestion temporal reasoning.

## Key Takeaways

| Observation | Evidence |
|-------------|----------|
| **6-network typed extraction is the strongest contribution** | All recalled facts correctly typed (experience, world, opinion, etc.); 2-pass extraction captures brief user mentions missed by single-pass |
| **Generation with Minimax-M2.7 + chain-of-thought solves temporal ordering** | c009 passed after model switch; `<think>` blocks show explicit "larger N weeks = earlier" reasoning |
| **Knowledge-update success requires session timestamps in generation context** | c005 ✓ (ISO timestamp visible) vs c006 ✗ (bare text, no timestamp) |
| **Temporal absolute date awareness is a systematic gap** | c011 and c010 both fail because "today"-relative expressions in memory become stale |
| **Recall breadth at top_k=25 is sufficient for most cases** | 8 of 12 cases had complete top-25 recall; failures concentrated in multi-session aggregation (c004, c010) |
| **Raw_snippet not decision-critical** | Across all analyzed cases, key facts were in the `text` field; raw_snippets were not needed for any correct answer |
