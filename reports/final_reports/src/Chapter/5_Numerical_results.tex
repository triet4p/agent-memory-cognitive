\subsubsection{Evaluation strategy}

Metric chính: End-to-end accuracy (LLM-as-judge), phân tích per-category. Cùng judge setup với HINDSIGHT để so sánh trực tiếp với published numbers.

\textbf{[FIGURE PLACEHOLDER]}\\
Hình 5.3: Evaluation pipeline end-to-end. Cần hình minh họa flow: Build KG → Recall → Reflect → Judge.

Các bước evaluation:
\begin{enumerate}
    \item Build CogMem graph offline một lần per conversation (cached, reused cho tất cả questions)
    \item Recall: Adaptive Router → 4-way retrieval → RRF → reranker → top-k nodes
    \item Reflect (stripped-down CARA): retrieved nodes + question → LLM generate answer
    \item Judge: LLM-as-judge(question, gold answer, generated answer) → binary score
\end{enumerate}

\subsection{Ablation study (E1-E7)}

\begin{enumerate}
    \item \textbf{E1 — Baseline}: HINDSIGHT rebuilt với SLM local. Điểm tham chiếu.
    \item \textbf{E2}: E1 + Habit Network + S-R links. Hypothesis: cải thiện Preference category.
    \item \textbf{E3}: E1 + Intention Network + Transition edges. Hypothesis: cải thiện Prospective queries.
    \item \textbf{E4}: E1 + Action-Effect Network + A-O causal links. Hypothesis: cải thiện Causal category.
    \item \textbf{E5}: E1 + Adaptive Query Router (incl. Causal + Prospective types). Hypothesis: cải thiện Temporal + Multi-hop.
    \item \textbf{E6}: E1 + Episodic Buffer SUM + 3 Cycle Guards. Hypothesis: cải thiện Multi-hop + Multi-session.
    \item \textbf{E7 — Full CogMem}: E1 + tất cả. Tổng hợp — so sánh chính.
\end{enumerate}

Các failure modes được phân tích:
\begin{enumerate}
    \item \textbf{Lossy compression residual}: Một số trường hợp raw\_snippet vẫn không đủ chi tiết
    \item \textbf{Mis-classified queries}: Adaptive router đôi khi misclassify query type
    \item \textbf{SUM over-accumulation}: Một số trường hợp tích lũy quá nhiều noise
    \item \textbf{Missing transitions}: Một số intention → experience transitions không được capture đúng
\end{enumerate}
